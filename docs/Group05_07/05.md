## Issues
* **Issue 1:** The IDataStructure interface was becoming too difficult to maintain and extend due to the different requirements of various data structures.
  * **Resolution:** Refactored the IDataStructure interface into three seperate interfaces: IStandardStructure for basic data structures like arrays and trees, IHeapStructure for heap-specific operations, and IGraphStructure for graph-specific operations. This seperation allows for better organization, easier maintenance, and more flexibility in implementing different types of data structures without affecting unrelated ones. Refactored the existing codebase to implement the new interfaces and ensure that all data structures are properly categorized and implemented according to their specific requirements. Additionally, refactored the Frame class to handle the more complex data structures requirements.
* **Issue 2:** Overflow issues and compatibility problems arose from using int for indices in the codebase.
  * **Resolution:** Strictly enforced using size_t for all indices in the codebase to prevent overflow issues and ensure compatibility. Refactored all existing code to replace int with size_t for indices.
